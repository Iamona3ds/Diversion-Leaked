WinDivert

    What is WinDivert

    WinDivert is a Windows packet interception driver and API that allows applications to capture, filter, block, inspect, and modify network traffic at a low level.

    How To Use

    This setup covers configuring Diversion with a WinDivert-based interception layer to handle Roblox client traffic at the network level. This approach allows modification of client settings responses by routing and processing traffic through the local interception system, without relying on external proxy tools. Once installed, ensure the interception service is running before launching Roblox. This allows the system to capture and process relevant requests during client startup.

    Diversion Proxy Setup

    Diversion integrates directly with the WinDivert-based interception system by loading a local JSON configuration file at runtime, matching Roblox ApplicationSettings requests, processing them through the internal network interception pipeline, and dynamically replacing or modifying responses without external proxy tools. This system works by intercepting Roblox network traffic before the client receives configuration data from Roblox servers. The application monitors outgoing HTTPS requests and checks whether the requested URL matches the Roblox ClientSettings endpoint: clientsettingscdn.roblox.com/ When a matching request is detected, the interceptor can redirect, replace, modify, or locally serve a custom response instead of the original server response. This allows custom fastflags and client configuration overrides to be applied dynamically at runtime. The interception layer is commonly implemented using a local HTTPS proxy and/or packet filtering drivers such as WinDivert. Since Roblox traffic is encrypted with HTTPS, direct packet editing alone is not sufficient; a proxy or TLS interception layer is typically required to inspect and modify the decrypted traffic.

    Similar to Fleasion, This approach operates at the network layer and modifies clientsettings apis in real time before the client processes the data.

    Diversion Setup

    1
    Run Diversion

    Self Explanatory

    Download Diversion Lastest

    2
    Generate the CA Certificate
    • Open the application and navigate to:
    • Menu → Proxy → Generate CA Certificate

    This creates a local Certificate Authority (CA) certificate that will be used for HTTPS interception.

    3
    Install the CA Certificate
    • After generating the certificate:
    • Menu → Proxy → Install CA Certificate

    This installs the certificate into Windows so HTTPS traffic can be decrypted locally.

    4
    Install the Certificate to Roblox
    • Next, install the generated CA certificate into Roblox's certificate bundle:
    • Install CA to Roblox cacert.pem

    This allows the Roblox client to trust the local interception certificate.

    5
    Launch Roblox Once
    • Open Roblox normally and allow it to fully launch.

    This lets Roblox initialize using the updated certificate configuration.

    6
    Close Roblox
    • Completely close Roblox before starting packet interception.
    • Make sure all Roblox processes are closed from Task Manager if necessary.
    7
    Start Network Capture
    • Enable the proxy or packet capture system:
    • Start Capture

    At this point the interceptor will begin monitoring Roblox traffic and checking requests against the configured URL filters.

    8
    Launch Roblox Again
    • Reopen Roblox after capture has started:

    The application should now intercept matching requests and apply custom responses or fastflag overrides in real time.

    Important Notes
    • If HTTPS interception fails, regenerate and reinstall the CA certificate.
    • Restarting Roblox is required after certificate changes.
    • Check that no VPN, proxy, or firewall is interfering with packet interception.
    • Verify that the CA certificate is properly installed and trusted in both Windows and Roblox's cacert.pem.
    • If fastflags do not apply, fully close Roblox and restart both the client and capture process.
    • Some antivirus software may block driver loading or certificate injection, so exclusions may be required.
